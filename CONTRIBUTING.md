## 拿幸教的贡献哲学

拿幸教不是一个严肃的项目，而是一个"邪教中的邪教"。所以我们的贡献指南也很"邪教"：

- **想加啥就加啥**——没有什么是不能加的
- **Prompt烂了？删了重写**——没有什么是不能删的
- **代码跑不通？无所谓，一切皆空**——没有什么是必须完美的
- **人人都是教主**——你的想法和贡献和教主的一样重要
- **逐教主出教门是你的权利**——不同意可以直接反对

**唯一的要求**：提交PR前记得加一句 **"Everything both nothing！"**

## 贡献方式

### 方式一：提交新的Prompt

你有一个新的拿幸教Prompt想法？直接提交！

**步骤**：
1. Fork本仓库
2. 在 `prompts/` 目录下创建新文件，格式：`N-拿幸[功能名].md`
3. 按照 [拿幸私人顾问](prompts/1-拿幸私人顾问.md) 的格式写（包括使用说明、Prompt模板、示例）
4. 提交PR，标题：`feat: 新增拿幸[功能名]Prompt`
5. **PR描述最后加上**：`Everything both nothing！`

**Prompt模板**：
```markdown
# 拿幸[功能名] Prompt

> **板块定位**：[简单说明这个Prompt的用途]

---

## 使用方法

[说明怎么用]

---

## Prompt模板（复制即用）

[粘贴Prompt代码]

---

## 示例

[提供1-2个示例]
```

### 方式二：改进现有Prompt

觉得某个Prompt不够好？直接改！

**步骤**：
1. Fork本仓库
2. 编辑你想改进的Prompt文件
3. 提交PR，标题：`improve: 改进拿幸[功能名]Prompt`
4. **PR描述最后加上**：`Everything both nothing！`

### 方式三：提交拿幸故事/金句/梗

你有好的拿幸故事、金句或梗想分享？

**步骤**：
1. Fork本仓库
2. 在 `docs/` 或相应目录下创建文件，或直接在Issue里提交
3. 提交PR或Issue，标题：`docs: 新增拿幸[内容类型]`
4. **描述最后加上**：`Everything both nothing！`

### 方式四：提交Meme或视频脚本

你生成了好的拿幸教Meme或视频脚本？

**步骤**：
1. Fork本仓库
2. 在 `assets/` 或 `docs/` 下创建文件夹存放
3. 提交PR，标题：`assets: 新增拿幸[内容]`
4. **PR描述最后加上**：`Everything both nothing！`

### 方式五：改进代码或脚本

你发现了bug或想优化脚本？

**步骤**：
1. Fork本仓库
2. 编辑 `scripts/` 下的Python文件
3. 提交PR，标题：`fix: 修复[问题]` 或 `refactor: 优化[功能]`
4. **PR描述最后加上**：`Everything both nothing！`

### 方式六：翻译或本地化

想把拿幸教传到其他国家？翻译一下！

**步骤**：
1. Fork本仓库
2. 在根目录创建 `i18n/` 文件夹
3. 创建 `[语言代码]/` 子文件夹（比如 `en/`、`ja/`、`ko/`）
4. 翻译相应文件
5. 提交PR，标题：`i18n: 添加[语言]翻译`
6. **PR描述最后加上**：`Everything both nothing！`

## 提交PR的检查清单

在提交PR前，请检查以下几点：

- [ ] 我的代码/内容符合拿幸教的风格（荒诞、幽默、有哲理）
- [ ] 我的PR标题清晰明了
- [ ] 我的PR描述说明了改动的原因
- [ ] 我在PR描述最后加上了"Everything both nothing！"
- [ ] 我没有违反任何法律或伦理（虽然拿幸教很"邪教"，但我们还是要守法）
- [ ] 我的内容不包含真实的个人隐私信息
- [ ] 我已经测试过我的代码/Prompt（如果适用）

## 代码风格指南

### Python代码

```python
# 好的拿幸教代码风格：
# 1. 有注释，但注释也要幽默
# 2. 变量名可以用中文，但要清晰
# 3. 函数要有docstring
# 4. 代码要能运行，但不一定要完美

# 示例：
def get_today_challenge():
    """
    获取今天的拿幸挑战
    这个函数会返回一个随机的挑战
    完成它，你就离极乐净土又近了一步
    """
    challenges = ["...", "..."]
    return random.choice(challenges)
```

```markdown
# 标题要清晰

**加粗**用来强调关键词

- 列表可以用来组织信息
- 但不要过度使用

[链接](url)可以指向其他资源

| 表格 | 也 | 可以 |
|------|-----|------|
| 用   | 来 | 组织 |
```

## 常见问题

### Q：我的想法/代码不够好怎么办？

**A**：没关系！拿幸教欢迎所有想法。一切皆空，所以"不够好"也是空的。提交吧，我们一起改进。

### Q：我的PR被拒绝了怎么办？

**A**：可能是因为：
1. 内容不符合拿幸教的风格
2. 内容违反了法律或伦理
3. 内容和现有内容重复
4. 我们暂时没有时间审核

不管是哪个原因，都不是对你的否定。Everything both nothing！

### Q：我可以删除别人的内容吗？

**A**：可以！这是拿幸教的特色。如果你觉得某个内容不好，可以提交PR删除它。但请在PR描述里解释为什么要删除。

### Q：我可以修改别人的Prompt吗？

**A**：可以！但请在PR里清楚地说明你的改进。原作者可能会有意见，但这也是讨论的一部分。

### Q：拿幸教有Code of Conduct吗？

**A**：有，就一条：**别伤害别人，别违法，别太较真。** 其他的都可以商量。
